Structs, Alignment, and Padding

Sections 1-7 of this webpage do an excellent job of explaining structure alignment and padding in C. Note, in all gcc versions 4.8+, struct reordering is no longer performed by the compiler. Structs will be laid out in memory as they are declared by the programmer. This means that the programmer has to be careful to not waste space!

To summarize the main takeaways:

  • Fields in a struct are laid out in memory in the order specified in the struct.
  • Each primitive data type must be placed in memory at an address which is a multiple of its size. IE. char 1 byte alignment, short 2 byte alignment, int 4 byte alignment, etc.
  • Padding is inserted between fields based on the required alignment.
  • In addition to the above, a struct must be aligned to the alignment of its largest field.

Memory Visualization and Access with Structs

Structs can be useful for more than solely creating grouped data types. They can also be used to "visualize"  and access data in memory through address casting with pointers. When a struct is cast onto a memory address, the struct definition acts as a template to access the data through the fields of the struct.

To use this method, define a pointer of the struct type. Assign the pointer to reference the starting memory location for which you want to access (to read or assign data to). Then using the pointer, access the fields of the struct just as you would if you had declared an instance of the struct.

Examine memory_access.c. In this file, 7 different structs are defined. Initialize value on line 40 of the main program with the value: unsigned long var = 0xBEEFCAFE

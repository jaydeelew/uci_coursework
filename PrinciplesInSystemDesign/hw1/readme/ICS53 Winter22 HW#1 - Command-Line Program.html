Assignment Overview

NAME

53secret - a program that “hides” a message into the provided text.

SYNOPSIS

53secret -L | -W MESSAGE [-S NUM] [-O] [-P]

DESCRIPTION

53secret reads characters from the standard input stream (STDIN) and based on the specified flag, "hides" the argument MESSAGE into the input text. The program prints information to the standard output stream (STDOUT) and standard error stream (STDERR). 

REQUIRED OPTIONS - Only one option can be specified at a time

-L MESSAGE        

Each of the alphabet letters in MESSAGE are "hidden" in the text read from STDIN in the order of appearance. If the character read from STDIN matches the current character to "hide" in MESSAGE, the character is printed to STDOUT as a capital letter. Otherwise, the character read is printed to STDOUT in lowercase.

Any non-alphabet characters in MESSAGE are ignored. Space characters in MESSAGE are ignored unless the -S option is specified.

The number of characters of MESSAGE "hidden" is printed to STDERR at the end of program execution.

-W MESSAGE        

A word is defined as a sequence of alphabetic & symbol characters delimited by whitespace. When each word in MESSAGE, in order of appearance, is found in the text read from STDIN, the line number (starting with 1) and the word position on the line (starting with 1) are printed to STDOUT in the format "LINENUM:WORDNUM\n".

Only full word matches are permitted, no substring matches. Each word of MESSAGE is matched with leading and ending punctuation included unless the -P option (Extra Credit) is specified.

The number of words of MESSAGE "hidden" is printed to STDERR at the end of program execution.

OPTIONAL OPTIONS - Only appear after the required option, in any order

   -S NUM

Can only be used with the L option. "Hides" the space characters in MESSAGE within the text read from STDIN. To "hide" the space character, NUM space characters are printed to STDIN after the next sentence punctuation character (period, comma, exclamation mark, or question mark) found in STDIN.

-O

When the capital letter o is used with the L option it enforces at most One character of MESSAGE to be embedded per line of input from STDIN. When used with -W option, it enforces at most One word of the MESSAGE to be embedded per line of input.

-P

[Extra Credit] Can only be used with the W option. Leading and trailing punctuation (any ASCII character between [0x21 '!', 0x7E '~'] except alphabet characters) are ignored on words in MESSAGE and when matching to words in STDIN text.

EXIT STATUS

The exit code indicates whether your program was able to successfully perform the requested operation, and if not, what kind of error occurred.

-1        Invalid program arguments, eg. invalid or bad argument value specified, an invalid option, no option, or too many options specified.  Usage statement prints to the standard error stream.

 0        Successful termination

 1        Partial MESSAGE successfully "hidden" in text.

A third approach to providing input to the 53secret program is through direct user input.  If the program is run without redirection, it will read from stdin (user input on the keyboard). As the user types input, the program will receive the input and process it (including newlines). When the program processes the input (char-by-char, per line, etc) it will output to stdout. Upon receipt of this signal, the terminal notifies the application that the end of the file has been reached (just like if it was reading from a file and had passed the last byte). Hints & Guidelines

  • DO NOT read STDIN entirely into memory. This is a waste of space! What if the input is GB or TB large? Your program should work for any size input. The best way to handle any size input for this type of program is to read the input character-by-character or line-by-line, process it, and then output it (if needed).
  • It is hard to see the different whitespace characters in a text editor or when printed to the terminal. To see the byte-by-byte contents of a file using the command-line program octal dump (man od). Here is a blog post describing a basic use that you can use.
  • Gradescope will use the diff program (man diff) to compare the output your program creates with the expected output. This means each and every byte matters! Each byte in the input should produce 0 or 1 byte in the output depending on the character. If you print extra ASCII characters or newlines to the output, this modifies the output! Use the diff (man diff) program to check your own inputs/outputs locally.  


Why does this matter? Command-line programs are often strung together using the pipe command. If you add extra characters/print statements to the output then you are modifying the information that is passed to the next program. It can impact their behavior.  We will see more of this in the next homework.

🤔What is a Makefile? 

Make is a program-building tool. It aids in simplifying building program executables that may need various modules. To determine how the modules need to be compiled or re-compiled together, make takes the help of user-defined makefiles.  A makefile is a specially formatted file composed of a series of commands. This file can be interpreted by make and make will execute.

Why do we need Makefile?

Compiling the source code files can be tiring, and get complex, especially when you have to include several source files and type the compiling command every time. Makefiles are the solution to simplify this task.

The main advantages are:

  • less typing and less error-prone compilation
  • recompilation is faster, as only the changes are compiled.
  • it makes it easier for other people to compile your code, as the compilations steps are outlined

It is not a requirement for your assignments to create your own Makefile to compile your programs. We will provide them for you. For those who are interested in learning about Makefiles, we will use different features or styles to create the Makefile with each assignment. We will also leave comments in the Makefile to explain how it works. We hope you can use them as a reference for learning.

Minimum Block Size

Read the code, compile, and run the program test_sizes1.c. This program calls the icslab_malloc for different primitive data types. After each call, the program will print out the state of the allocator (in blocks). Each block is printed with the relevant information, include block size, allocated or free, and the addresses associated with the header and the footer of each block.  The program will pause waiting to take input from the user before continuing. Run the program bin/test_sizes1.out

Remember, the allocated bit is the least significant bit of the block size field. When a block is allocated the bit is set to 1. Therefore, the value of the blocksize field will be an odd number and the value will be 1 byte greater than the actual size. When a block is free, the bit is set to 0. Therefore, the value of the blocksize field will be an even number and the value will be exactly the block size.

Multi-page Allocation

The allocator requests space on the heap as necessary. The heap will return 1 page of space (4096 bytes) with each request. In the lab and homework code, a maximum of  4 pages can be requested.

When a malloc request exceeds the available memory space managed and currently free, the allocator will try to request more space from the heap. If it is available, the allocator will join this new space to the space it currently manages and then tries to satisfy the request.

Examine the code in test_exceed1pg.c and the run bin/test_exceed1pg.out

Answer Question 6 on the Canvas Quiz.

Coalescing

When a block is free'd it can be joined with the physically adjacent blocks in memory which are also free. This allows free space to be coalesced (joined together) to build larger blocks of memory space to satisfy new malloc requests. There are 4 different coalescing scenarios. Examine the code in test_coalesce_case*.c and then run each of them to visualize how coalescing works.

Case 1: both adjacent blocks are allocated.

Case 2: adjacent block at higher memory address is free.

Case 3: adjacent block at lower memory address is free.

Case 4: both adjacent blocks (higher and lower memory addresses) are free.

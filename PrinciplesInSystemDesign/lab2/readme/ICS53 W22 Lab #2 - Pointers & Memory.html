While the program is paused, suspend this foreground process by pressing Ctrl-Z. This will place the foreground process into the background and allow the bash shell to take a new command. This is a type of process management. We will study this in more detail as our next topic.

While the program is running we can visualize the memory layout of the program using the files in the proc filesystem (man proc). If you list the contents of the /proc directory (ls /proc), you will see a lot of files. Our focus is on one of them: /proc/[pid]/maps, where [pid] is replaced with the pid (unique process id)  our current running program. A process is an instance of a program. According to the man page, maps is  a "file containing the currently mapped memory regions and their access permissions".

To see the regions for our program, we first need to determine its pid. We can use the program ps to get the PID of a running process (man ps).

Run the following command to list all the currently running processes in your system:

$ ps aux

When doing so you can see the column labeled PID and the column COMMAND. These are the columns we need to find the PID of our currently running program.

Utilizing what we have learned: To only obtain the running processes with the name ./a.out, use the following set of commands (uses pipes)

$ ps aux | grep ./a.out | grep -v grep

In this case, the stdout of the program ps is provided to the stdin of grep. Then the stdout of grep is sent to grep again to remove the grep program from the list.

To view the memory addresses for each region we can examine the file /proc/[pid]/maps. Open the file in vi or nano or use the program cat to print the contents to the terminal. This will display the address range for each region and the permissions as well as a few other pieces of information (man proc for more info).

Your output should look similar to this where 12345 is the PID of the program:

$ cat /proc/12345/maps

00400000-00401000 r-xp 00000000 fd:00 17184229                           /home/vagrant/a.out

00600000-00601000 r--p 00000000 fd:00 17184229                           /home/vagrant/a.out

...

Dynamic Memory Allocation

Often the size of the data structures in your program is not known ahead of time (think array-lists, linked-lists, trees, etc.). Dynamic memory allows for allocation of memory space as needed, a.k.a, dynamically. While we will study how to manage dynamic memory using an allocator later this term, the goal now is to solely understand how to use dynamic memory.

The C programming language provides dynamic memory allocation through the malloc function. However, unlike many modern programming languages, the memory is not released automatically. The programmer needs to be sure to call free to return the dynamically allocated memory once it is no longer needed.
